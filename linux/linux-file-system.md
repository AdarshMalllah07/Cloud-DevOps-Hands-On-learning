# 1. What is a File System?

A File System is the method used by Linux to store, organize, retrieve, and manage files on storage devices.

Think of it like a library.

Without a file system:

- Linux cannot locate files.
- Applications cannot read or write data.
- Permissions cannot be enforced.

A file system keeps track of:

- File names
- Locations on disk
- Ownership
- Permissions
- Metadata

---

# 2. Linux File System Hierarchy

Unlike Windows, Linux does not use C:, D:, or E: drives.

Linux has only one starting point:

```

/

```

Everything begins from the Root Directory.

```

/
├── bin
├── boot
├── dev
├── etc
├── home
├── lib
├── media
├── mnt
├── opt
├── proc
├── root
├── run
├── sbin
├── srv
├── sys
├── tmp
├── usr
└── var

```

This structure is known as the **Filesystem Hierarchy Standard (FHS)**.

---

# 3. Root Directory (/)

The Root Directory is the top-level directory in Linux.

Every file and directory exists under it.

Examples:

```

/home/adarsh

```

```

/etc/nginx

```

```

/var/log

```

Every absolute path starts with `/`.

---

# 4. Important Directories

## /bin

Contains essential user commands required by the operating system.

Examples:

- ls
- cp
- mv
- rm
- mkdir
- pwd
- cat

Check command location:

```bash
which ls
```

Example Output:

```text
/usr/bin/ls
```

Modern Linux distributions merge `/bin` into `/usr/bin`, but conceptually `/bin` contains essential binaries.

---

## /etc

Stores system-wide configuration files.

Examples:

```
/etc/passwd
/etc/hosts
/etc/fstab
/etc/ssh/
/etc/nginx/
/etc/systemd/
```

Examples:

SSH Configuration

```
/etc/ssh/sshd_config
```

Nginx Configuration

```
/etc/nginx/nginx.conf
```

Rule:

> Configuration files belong in `/etc`.

---

## /home

Stores personal files for normal users.

Example:

```
/home/adarsh
```

Contains:

- Documents
- Downloads
- Desktop
- Projects

Every normal user has a separate home directory.

---

## /root

Many beginners confuse `/` with `/root`.

```
/
```

means Root Directory.

```
/root
```

means Home Directory of the root user.

Only the root user normally works here.

---

## /var

Contains variable data.

Examples:

```
/var/log
/var/cache
/var/lib
/var/tmp
```

### /var/log

Stores application and system logs.

Examples:

```
/var/log/syslog
/var/log/auth.log
/var/log/nginx/
```

Whenever a service crashes, DevOps Engineers usually check logs here.

---

## /tmp

Stores temporary files.

Applications use this location for temporary data.

Contents may be cleared after reboot.

Example:

```
/tmp/upload.tmp
```

---

## /usr

Contains user applications, libraries, documentation, and binaries.

Examples:

```
/usr/bin
/usr/lib
/usr/share
```

Programs like Python, Git, Vim, Curl, etc., are stored here.

---

## /opt

Stores optional third-party software.

Examples:

```
/opt/google
```

```
/opt/sonarqube
```

---

## /boot

Contains files required during the Linux boot process.

Includes:

- Linux Kernel Image
- GRUB Bootloader
- Initramfs

Without this directory, Linux cannot boot.

---

## /dev

Linux treats hardware devices as files.

Examples:

```
/dev/sda
/dev/null
/dev/random
/dev/tty
```

Hard Disk → File

USB → File

Terminal → File

Everything is represented as a file.

---

## /proc

A virtual filesystem generated by the Linux Kernel.

It does not exist physically on disk.

Examples:

CPU Information

```bash
cat /proc/cpuinfo
```

Memory Information

```bash
cat /proc/meminfo
```

Running Processes

```
/proc/1
```

Everything is generated dynamically.

---

## /sys

Another virtual filesystem.

Used by:

- Kernel
- Drivers
- Hardware Management

Mostly used for low-level system configuration.

---

# 5. Real World Cloud Example

Suppose your AWS EC2 instance is running Nginx.

Configuration:

```
/etc/nginx/
```

Logs:

```
/var/log/nginx/
```

Binary:

```
/usr/sbin/nginx
```

Website Files:

```
/var/www/html/
```

When troubleshooting, you already know where everything is stored.

# 6. Hands-on Commands

Show Current Directory

```bash
pwd
```

List Root Directory

```bash
ls /
```

List Configuration Files

```bash
ls /etc
```

List Home Directories

```bash
ls /home
```

List Variable Data

```bash
ls /var
```

View CPU Information

```bash
cat /proc/cpuinfo
```

View Memory Information

```bash
cat /proc/meminfo
```

Display Directory Structure

```bash
tree -L 1 /
```

If tree is not installed:

```bash
sudo apt install tree
```

# 8. Revision Table

| Directory | Purpose |
|------------|---------|
| / | Root Directory |
| /bin | Essential binaries |
| /etc | Configuration files |
| /home | User home directories |
| /root | Root user's home |

| /tmp | Temporary files |
| /usr | User applications |
| /opt | Optional software |

| /dev | Device files |
| /proc | Virtual Kernel information |

---

# 9. Key Takeaways

- Linux follows a single-root filesystem.
- Every file starts from `/`.
- Configuration files are stored in `/etc`.
- Logs are stored in `/var/log`.
- User files are stored in `/home`.
- Root user's files are stored in `/root`.
- Hardware devices are represented as files in `/dev`.
- `/proc` and `/sys` are virtual filesystems generated by the Kernel.
